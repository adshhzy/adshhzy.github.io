1. Algorithm for texture synthesis:  Basically follows the core algorithm described in the paper.
          1) Using SSD of overlapping to find the suitable subimages orderly from left to right  each row and from top to bottom. In each position, randomly cut 1000 samples and pick                 the one minimizes the overlapping SSD.  Store chosen subimages with the overlapping region.
          2) After finding all subimages, from  left to right, orderly merge the image with respect to each row using the dynamic programming method in the paper.
          3) From top to bottom, orderly merge rows using the dynamic programming method in the paper.

2.  Algorithm for texture Transfer: The same as the algorithm described in the paper, using the same pineline above with modified error function in each iteration.
